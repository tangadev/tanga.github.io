Authentication

We use http authentiction. Use the same credentials you use to login into the vendor portal https://vendors.tanga.com.

Vendor ID

Anywhere you see it is the vendor id which is the id in the URL when you are viewing your pages on the vendor portal https://vendors.tanga.com.

Inventory

[Manual CSV]
You can currently upload your inventory or get a listing of inventory in the Vendor Portal

[API]
inventory report in json format:
curl --user email:password https://vendors.tanga.com/api/vendors/inventory_report

update single product inventory:
curl --data "vendor_sku_code=<sku code>&in_stock=<in stock count>" https://vendors-dev.tanga.com/api/v1/drop_shippers/<id>/inventory

update bulk inventory:
curl -F "file=@<path to local inventory csv>" https://vendors.tanga.com/api/v1/drop_shippers/<id>/inventory/file

Upload Tracking

[API]
upload single tracking number:
curl -X POST -d package_id=12234 -d tracking_number=1Zasdf --user email:password https://vendors.tanga.com/api/vendors/<id>/set_tracking

upload multiple tracking numbers via csv file:
curl -X POST --user email:password -d "csv=cat /tmp/t.csv" https://vendors.tanga.com/api/vendors/<id>/set_tracking

The CSV file should be in the same format as the one you upload to https://vendors.tanga.com.

[Manual CSV]
You can manually upload tracking via CSV format LINK HERE in your vendor portal:
https://vendors.tanga.com/drop_shippers/<id>/purchase_orders/orders/csv

Orders

[API]
fetch unshipped orders in csv format:
curl --user email:password https://vendors.tanga.com/api/vendors/unshipped_items.csv
This will return the data in csv format you define.

fetch unshipped orders in json format:
curl --user email:password https://vendors.tanga.com/api/vendors/unshipped_items

Fetching unshipped items support start_at and end_at iso8601 parameters for fetching a range of orders.
?start_at=2015-05-03T15:39:03+00:00&end_at=2015-05-04T15:39:03+00:00

[Manual CSV]
You can manually download orders via CSV format in your vendor portal:
https://vendors.tanga.com/drop_shippers/<id>/purchase_orders
